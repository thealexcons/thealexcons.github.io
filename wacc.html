WACC Compiler

WACC is a simple imperative and statically-typed programming language, supporting a variety of common features and constructs. In a team of four students, we built a compiler from scratch using Scala that produces ARM assembly code from WACC source code. We decided to use a cutting-edge parser combinator library for Scala on the frontend and wrote the code generator and semantic analysis by hand. The compiler will also output any compilation errors such as syntax or semantic errors.
As an extension to the original language specifications, we added a range of new features, including simple classes with fields and methods, interfaces and generic type parameters for functions.

Some of the main language features are:

  • Classes with public/private fields and methods
  • Generic type parameters for (non-method) functions with type constraints
  • Interfaces (like in Java)
  • Loops
  • Arrays and pairs/tuples
  • Simple types: integers, booleans, strings and characters
  • I/O functions: printing and reading from/to stdout/stdin

Some example WACC programs can be viewed in this repository.

What I learnt:
  • What parser combinators are and how they can be used to build a fully-fledged parser
  • How semantic analysis works and all the different checks involved, including type and scope checks
  • How assembly code is generated by recursively traversing the AST of the program
  • How runtime checks work and how they are included in the generated assembly
  • A more detailed understanding of assembly code and invoking LibC functions using the Linux ABI
