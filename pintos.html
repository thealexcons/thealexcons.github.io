Pintos: Simple x86-based Operating System

The goal of this project was to implement the core components of an operating system's kernel, including a thread scheduler to support multi-threading, loading & executing user programs, a set of important system calls and a virtual memory system, which was built from scratch.

The operating system was implemented in C by a team of four (including myself) during the first term of my second year at Imperial College London. The project included several code reviews with PhD students and written documents describing our design choices and implementation.

The first part of the project was to implement a thread scheduler. We had to implement a priority scheduleling algorithm with priority donation/inheritance as well as a multi-level feedback queue scheduler. The second task consisted in implementing a set of system calls (which include filesystem interaction) and allowing the execution of user-supplied programs, including parsing command line arguments and building the program's stack frame.
The final part was to build a virtual memory system entirely from scratch, using paging. This involved evicting frames to disk when memory was full, sharing common frames among different programs, stack growth and memory-mapped files.

What I learnt:

  • Reading through a large existing codebase and its documentation to understand a software system
  • Kernel development, giving me a deeper insight into modern operating systems and their internal workings
  • Putting into practice many theoretical OS concepts, such as scheduling algorithms and virtual memory
  • Hugely improved my debugging skills for concurrency-heavy code
  • Considering and evaluating different design choices for implementation and discussing their trade-offs with fellow peers
